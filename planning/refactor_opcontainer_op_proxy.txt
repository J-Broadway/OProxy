# Refactor_OPContainer_OP_Proxy.txt CURRENTLY TESTING

## OVERVIEW
This document outlines the refactoring plan to transform OProxy into a true dynamic metaprogrammatic proxy system for TouchDesigner by implementing a hybrid Proxy-Container design pattern. The goal is to enable `opr.Test('moviefile1')` to simultaneously function as both an OP_Proxy (for single OP access) and an OPContainer (for managing child OPs).

## CURRENT ARCHITECTURE ANALYSIS

### Current State
- **OP_Proxy**: Wraps individual TouchDesigner OPs, inherits from OPBaseWrapper
- **OPContainer**: Manages collections of OP_Proxy instances, inherits from `list` and OPBaseWrapper
- **opr**: Root class that creates dynamic OPContainer subclasses via metaprogramming
- **Storage**: Hierarchical storage system with OPs, Extensions, and Children nodes

### Current Limitations
1. `opr.Test('moviefile1')` returns OP_Proxy, not OPContainer
2. OP_Proxy cannot contain child OPs
3. No context-aware delegation between proxy and container behavior
4. Extensions on individual proxies don't persist properly

## TARGET ARCHITECTURE

### Core Design Pattern: Hybrid Proxy-Container
- **Single Object Type**: OPContainer that can contain 0, 1, or multiple OPs
- **Context-Aware Behavior**: Automatically switches between proxy and container modes
- **Unified Interface**: Same object can be accessed as both proxy and container

### Behavioral Rules
- **len(self) == 0**: Empty container behavior
- **len(self) == 1**: Proxy behavior (delegates to single OP)
- **len(self) > 1**: Container behavior (manages multiple OPs)

## IMPLEMENTATION PLAN

### Phase 1: Core OPContainer Refactoring

#### 1.1 Modify OPContainer.__getattr__()
**File**: `OPContainer.py`
**Current**: Delegates all attributes to child OPs
**New**: Context-aware delegation

```python
def __getattr__(self, name):
    # Check for container-level attributes first
    if hasattr(self.__class__, name):
        attr = getattr(self.__class__, name)
        if hasattr(attr, '__get__'):
            return attr.__get__(self, type(self))
        return attr
    
    # Context-aware delegation based on container size
    if len(self) == 1:
        # Single OP - proxy behavior
        if hasattr(self[0].op, name):
            return getattr(self[0].op, name)
        # Fall back to OP_Proxy behavior
        return getattr(self[0], name)
    else:
        # Multiple OPs - container behavior
        return [getattr(w, name) for w in self]
```

```python
                'Children': {}
            }
        }
    }
}
```

### Storage Changes Required
1. **Node Creation**: Support empty containers

3. **Extension Storage**: Ensure single-item containers can store extensions
4. **Traversal**: Update tree traversal for hybrid objects

## API COMPATIBILITY

### New Capabilities
```python
# Create container with single OP
opr._add('Test', ['moviefile1'])

# Access as proxy
opr.Test('moviefile1').par.width = 1920
opr.Test('moviefile1').path

# Access as container
opr.Test('moviefile1')._extend('another', ['moviefile3', 'moviefile4'])
for i in opr.Test('moviefile1').another:
    print(i.path)

# Mixed usage
opr.Test('moviefile1').par.width = 1920  # Proxy behavior
opr.Test('moviefile1')._add('more', ['moviefile5'])  # Container behavior
```

### Backward Compatibility
- Existing code using OPContainer continues to work
- Existing code using OP_Proxy needs migration
- Extension system remains compatible
- Storage format remains compatible

## CONCLUSION

This refactoring will transform OProxy into a true dynamic metaprogrammatic proxy system that provides maximum flexibility while maintaining backward compatibility. The hybrid Proxy-Container pattern enables seamless switching between proxy and container behavior based on context, making the system more intuitive and powerful for TouchDesigner developers.

The implementation is complex but achievable, requiring careful coordination across multiple files and systems. The phased approach ensures minimal disruption while delivering the full vision of a dynamic, context-aware proxy system.
